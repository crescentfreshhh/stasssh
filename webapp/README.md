# Opus megaboard

A static web page: a grid of looping `<video>` tiles, each playing an **apex**
segment live-streamed from Stash, swapping in a new one as it ends — cycling
continuously until you pause.

No build step, no framework. It reads `playlist.json` (generated by
`peaks playlist`) in which the Stash stream URLs are already baked, so the page
never talks to the Stash GraphQL API — it just plays video.

## Run it

```bash
peaks score --write     # make sure some apex markers exist in Stash
peaks playlist          # writes webapp/playlist.json
peaks serve             # serves this folder at http://localhost:8800
```

Open the URL. `playlist.json` is gitignored (it contains stream URLs with your
API key).

## Controls

- **grid** — 2×2 … 5×5. Keep it modest (3×3) for live streaming so you don't
  overload Stash's transcoder with too many simultaneous seeked streams.
- **Pause / Play** — stop/▶ all tiles and the cycling.
- **Reshuffle** — reload every tile with a fresh apex.
- **Muted / Audio on** — tiles start muted (browsers require it for autoplay).
  **Click any tile** to unmute just that one (click again to re-mute).

## How a tile cycles

Each tile streams its apex from `?start=`, advances when ~the segment's length
has played (or on `ended`/`error`), then the weighted picker hands it the next
apex — favouring higher-scored ones and avoiding immediate repeats.

## Notes / future

- Live-stream mode means every tile is a transcoded seek; smoothness depends on
  your Stash box. The planned **pre-cut clips** path (see `docs/ARCHITECTURE.md`)
  will make big grids buttery by serving short static files instead.
- `playlist.json` is regenerated each time you run `peaks playlist`, so re-run
  it after scoring new apexes.
